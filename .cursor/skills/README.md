# Cursor / agent skills (curated)

This directory replaces the former **`.claude/skills/`** tree. Only skills that support **workflow discipline** (plan, cook, fix, test, review), **CABTA-friendly engineering** (frontend/backend/web-testing), **MCP**, **GitNexus**, and **docs/git** are kept.

## Layout

- One subdirectory per skill (e.g. `plan/`, `cook/`, `gitnexus/`).
- Each skill typically has a **`SKILL.md`** (instructions for the AI); some include `scripts/` for optional CLI helpers.

## Conventions

- Paths in `SKILL.md` files use **`.cursor/skills/...`** (this repo).
- For CABTA product rules and read order, start with **`CABTA/docs/ONBOARDING.md`**, not only these skills.
